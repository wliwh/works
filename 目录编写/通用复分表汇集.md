# UDC通用复分号

- (0...)形式复分. 表1d

范围注释: 形式复分表用于表示文献的形式和外观，不表示文献主题。文学形式（诗歌、戏剧、小说等）归在82-1/-9。表1d未列出的形式类型可以用(0....)表示，例如，94(0:791)历史表示为一部电影。内在与外在形式。当需要表达多个形式因素时，通过内在形式和外在形式作出区分。内在形式是形式影响主题，例如，历史性表征；外在形式仅表达信息载体的物理特征，例如，录音。无论是否数字次序，内在形式应紧挨主题，出现在外在形式之前，例如，792(091)(086.7) 剧院 — 历史表现形式 — 录音（剧院录音的历史）应用注释: 引用次序：这些复分表通常位于主题类号之后，但是，如果需要的话，相同表现形式（或它们的替代物）的所有文档可以归类在适合的形式复分之下。合并示例:
```
- (035)54	化学手册（与其他手册并列）
- (038)54	化学字典（与其他字典并列）
- (054)(44)	法国报纸
54(035)	化学手册（与其他化学作品并列）

```

# 修订版通用复分表

设计说明

1. 前缀符号：使用 =（等号）。在大多数文件系统中，= 的排序优先级通常位于数字和字母之前，且在视觉上具有很强的“属性赋值”含义（例如 Programming=History）。
2. 通用性：本表设计时充分考虑了“社会科学”与“自然科学”的共性，既能处理计算机的代码，也能处理文学的作家传记。
3. 互斥性：尽量保证每个复分号的定义互斥，避免文件归类时的犹豫。
4. 移除了“=7 地区与国别”，并遵循“不强制填满数字”的原则，留出了逻辑空隙，使分类的认知边界更加清晰。

### 设计逻辑解析

1.  **=2 (教程) vs =3 (实务) 的核心区别**：
    *   **=2** 是**“从0到1”**：它是给**学生**看的，目的是让你学会。例如《Java编程思想》、《21天学通C++》。
    *   **=3** 是**“从1到N”**：它是给**工人/工程师**看的，目的是解决具体问题。例如《Spring官方文档》、《Java虚拟机规范》、《高性能MySQL实战》。
    *   *注：这种区分在计算机类电子书中至关重要，否则文件夹会非常混乱。*

2.  **=6 (社会) 的独立性**：
    *   将法律、伦理、组织单独列出。在AI时代，**AI伦理**、**开源协议**、**数据安全法**等内容越来越多，值得拥有一个独立的代码，而不是挤在理论里。

3.  **留白 (=7) 的意义**：
    *   跳过7，在视觉上将“文本/知识类资料 (0-6)”与“非书/历史类资料 (8-9)”隔开，减少了记忆负担。

### 设计细节QA

**Q1: 为什么把“研究(=4)”和“教育(=2)”分开？**
**A:** 在电子书管理中，这是一个巨大的痛点。**教科书(=2)**是系统化的、已确定的知识，适合初学者；而**论文与研报(=4)**是碎片化的、探索性的知识，适合专家。将两者混在一起会增加检索难度。

**Q2: “=7 地区”在计算机类中怎么用？**
**A:** 确实在纯技术领域用得少，但在宏观技术领域很有用。例如在 `半导体` 分类下，可以分为 `=7 中国芯片产业` 和 `=7 美国芯片产业`。如果该分类下不需要地理区分，则不使用此复分号。

**Q3: “=8 资源”与“=3 实务”的区别？**
**A:** **=3** 是**书**（如《如何写代码》），是给人看的指导；**=8** 是**数字资产**（如源代码本身、音频文件、软件安装包），是直接使用的对象或非文本媒体。

**Q4: 历史类书籍（如《二战史》）本身就是历史，还要加 =9 吗？**
**A:** 不需要。如果主分类已经是“历史”，则直接分类（如“太平洋战争”、“欧洲战场”）。但如果是“计算机”下的“历史”，则必须加 **=9**，变成 `计算机/=9 计算机发展史`。这就是“主干直接分，枝节用复分”的原则。
