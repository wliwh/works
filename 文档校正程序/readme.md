## `md_ai` 整合与优化说明

### 主要变更

1. 页面提取逻辑优化
    - 鲁棒性增强：现在 `extract_pages` 方法使用正则表达式提取页码，能够处理 `========404*****` 或 `========405=====` 等各种标记变体。
    - 起始页处理：修复了原版本在指定 start_page 时会收集该页码之前所有不包含标记内容的问题。现在程序会严格从指定的起始页开始收集内容。
2. CLI 命令行参数扩展
    - 为提升使用灵活性，新增了以下命令行参数：
        - --suffix: 指定输出文件的后缀（例如 --suffix _correct4）。
        - --marker: 指定页面分隔符（默认为 =========）.
        - --model: 指定 Ark API 的模型名称。
3. 功能补全
    - 标点符号映射：在 `PunctuationConverter` 中以注释形式保留了旧程序中出现的所有备选符号映射，方便后续根据需要启用。
    - 校验逻辑：保留并优化了对 AI 输出结果的“疑似截断”校验（即 】 结尾或长度异常检测）。

### 验证结果

通过测试脚本验证，程序能够正确识别并处理以下场景：

- 跳过起始页码之前的所有内容（包括未标记的文字）。
- 正确识别带星号（*）或等号（=）后缀的复杂页码标记。
- 正确处理 end_page 之后的截断，精准地停止读取并在停止前完整保留最后一页的内容

### 示例用法

```bash
python3 md_ai.py input.txt --mode page --start 403 --end 510 --suffix _correct4
```
